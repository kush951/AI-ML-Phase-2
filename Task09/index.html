How this check works

After each model retrain or pricing change, we run the best model's predictions over the full dataset and split results by payment_tier. If precision or recall gap exceeds 8 percentage points, the system flags a regression and pages the AI team. This guards against silent degradation where a paywall change subtly changes the candidate pool the model is evaluated on.

Failure Handling & Resilience

Every failure mode is handled explicitly — with real money, failure handling IS the feature

💳

Payment Failure Mid-Application

If payment fails after form submission, the student's data is held in a pending queue (48h TTL). Application slot is not counted. Student receives email with retry link. Money is NEVER deducted for a failed charge.

Handled ✓
🔄

Gateway Reconciliation Mismatch

Every 15 minutes a reconciliation job compares our DB records vs the payment gateway ledger. Discrepancies raise an alert in Slack and block further charges until resolved. We never trust only one source.

Handled ✓
🤖

Model Inference Failure

If the ML model throws an exception, the API falls back to the Baseline (Skill Overlap) rule. The UI shows "Match score unavailable — showing top skill matches" so students are never left with a blank screen.

Handled ✓
📉

Relevance Regression (Paywall Skew)

The conversion-quality check runs after every retrain. If precision/recall gap between paid and free exceeds 8 pp, deployment is blocked and the previous model is restored automatically from the model registry.

Handled ✓
⏱️

Test Mode vs Real Money

The PAYMENT_MODE env var controls test/live. All staging environments are hardcoded to test mode. A deployment checklist enforces: confirm PAYMENT_MODE=live before any prod release. CI blocks deploy if PAYMENT_MODE is ambiguous.

Pre-prod check ✓
🔌

Upstream Dependency (Baseline) Late

If the baseline dependency isn't ready by T-1h, the pipeline uses the last available baseline snapshot and flags the run as "provisional". The founder is notified and the run is re-evaluated once the dependency lands.

Monitored ✓
